2. Pull vs Push System

There are two things Data consumer and Data producer

Pull System - In Pull systems, the Consumer determines when it receives data from the data Producer. The Producer itself is unaware of when the data will be delivered to the Consumer.Like in javascript every function is pull system. Javascript function will never return data untill and unless it has been called. The code written for invoking function is data consumer and the function(return statement) is data producer


Push system - In this system data producer will determines when it has to send data to data consumer, like Promises in javascript

As promises either get resolved or reject it means promise return only single value

{{this.generatedValueByPromise != null ? this.generatedValueByPromise > 5 ? this.generatedValueByPromise + '- Promise gets rejected' : this.generatedValueByPromise +'- Promise gets resolved' : null}}

To see demo click above button it will generate any number between 1 - 10 if number is greater than 5 then our promise will get rejected and otherwise our promise will be resolved. Now let number is 5 so it will get resolved after 5 sec, let number is 3 it will get resolved in 3 sec. So here when button is clicked data consumer wants data but as Promises(data producer) will not send data immediatly, it will send data accordingly. So mainly Promises are used to perform or to evalute asynchronous data

Also Observables are also an important part of Push system
